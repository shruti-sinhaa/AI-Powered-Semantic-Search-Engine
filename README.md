# 🔍 Semantic Search Engine: 
BERT & Elasticsearch A high-performance semantic search system that moves beyond keyword matching to understand the contextual meaning of queries. This project leverages BERT (Bidirectional Encoder Representations from Transformers) to generate 768-dimensional dense vectors and Elasticsearch for efficient k-Nearest Neighbor (kNN) retrieval.

# 🚀 Features:
- Contextual Understanding: Uses sentence-transformers (BERT-based) to capture deep semantic relationships.
- Vector Database: Implements Elasticsearch's dense_vector field type with HNSW (Hierarchical Navigable Small World) indexing for sub-second search latency.
- Fast kNN Search: Uses the knn search API for approximate nearest neighbor retrieval.
- Scalable Pipeline: Includes a modular indexing script and a query interface for real-time search.

  # 🛠️ Tech Stack:
- Language: Python
- NLP Model: BERT (all-mpnet-base-v2 or bert-base-uncased) via Hugging Face.
-  Search Engine: Elasticsearch 8.x
-  Libraries: elasticsearch-py, sentence-transformers, pandas, torch.

 # 🏗️ System Architecture 
 1) Preprocessing: Raw text data is cleaned and tokenized.
 2) Embedding: Each document is passed through a pre-trained BERT model to generate a numerical vector (embedding).
 3) Indexing: Documents and their corresponding embeddings are indexed into Elasticsearch.
 4)  Querying: - User input is converted into a vector using the same BERT model.
 - Elasticsearch performs a cosine similarity search between the query vector and indexed document vectors.
 - Top $k$ most relevant results are returned based on the similarity score.

# 💻 Implementation Details: 
1. Elasticsearch Index Mapping We define the index with a dense_vector field. For a standard BERT base model, the dimensionality is 768.
 ```bash { "mappings": { "properties": { "text_content": { "type": "text" }, "vector_embedding": { "type": "dense_vector", "dims": 768, "index": true, "similarity": "cosine" } } } } ```
 2. Semantic Search Query The search is performed using the knn option to find the closest vectors in the latent space.
```bash search_query = { "field": "vector_embedding", "query_vector": query_vector, # Generated by BERT "k": 5, "num_candidates": 50 } ```
